# Примеры виртуальных пациентов

Эта папка содержит примеры клинических случаев для обучения и тестирования модуля виртуальных пациентов.

## Доступные случаи

### 1. Бронхиальная астма (`bronchial_asthma.json`)
- **Сложность**: 2/5
- **Специальность**: Пульмонология
- **Ключевые симптомы**: Одышка, кашель, свистящее дыхание
- **Учебные цели**: Дифференциальная диагностика обструктивных заболеваний легких

### 2. Острый аппендицит (`acute_appendicitis.json`)
- **Сложность**: 2/5
- **Специальность**: Хирургия
- **Ключевые симптомы**: Абдоминальная боль, тошнота, лихорадка
- **Учебные цели**: Диагностика острого живота, интерпретация перитонеальных симптомов

## Использование

### Загрузка случая в Python:

```python
import json
from pathlib import Path
from core.modules.virtual_patient.patient_model import VirtualPatient

# Загрузить случай
case_file = Path("core/modules/virtual_patient/examples/bronchial_asthma.json")
with open(case_file, 'r', encoding='utf-8') as f:
    case_data = json.load(f)

# Создать виртуального пациента
patient = VirtualPatient.from_dict(case_data)

# Получить базовую информацию
print(f"Пациент: {patient.name}, {patient.age} лет")
print(f"Жалобы: {patient.chief_complaint}")

# Раскрыть жизненные показатели
vital_signs = patient.reveal_info("vital_signs")
print(f"Витальные показатели: {vital_signs}")

# Провести поиск (симуляция заказа лабораторных анализов)
lab_result = patient.reveal_info("lab_results", "complete_blood_count")
print(f"Общий анализ крови: {lab_result}")

# Оценить диагноз студента
student_diagnosis = [
    {"name": "Бронхиальная астма, аллергическая форма, легкое персистирующее течение", "probability": 0.85}
]
evaluation = patient.evaluate_diagnosis(student_diagnosis)
print(f"Оценка: {evaluation['score']}/100")
print(f"Обратная связь: {evaluation['feedback']}")
```

## Создание новых случаев

Чтобы создать новый клинический случай:

1. Скопируйте один из существующих JSON файлов
2. Измените данные пациента:
   - Демографические данные
   - Анамнез и жалобы
   - Витальные показатели
   - Результаты обследований
   - Диагнозы с вероятностями
3. Добавьте экспертное объяснение в поле `expert_reasoning`
4. Укажите уровень сложности (1-5)

## Структура JSON файла

```json
{
  "patient_id": "Уникальный ID",
  "name": "Имя пациента",
  "age": возраст,
  "gender": "М/Ж",
  "chief_complaint": "Основная жалоба",
  "history_of_present_illness": "Анамнез заболевания",

  "vital_signs": {
    "pulse": { ... },
    "temperature": { ... }
  },

  "symptoms": [ ... ],
  "medical_history": { ... },
  "physical_exam": { ... },
  "lab_results": { ... },
  "imaging_studies": [ ... ],

  "diagnoses": [
    {
      "name": "Название диагноза",
      "probability": 0.0-1.0
    }
  ],

  "scenario_difficulty": 1-5,
  "expert_reasoning": "Детальное объяснение"
}
```

## Уровни сложности

- **1** - Простой случай с явной клинической картиной
- **2** - Типичный случай, требует базовых знаний
- **3** - Средняя сложность, требует дифференциальной диагностики
- **4** - Сложный случай с атипичными проявлениями
- **5** - Очень сложный случай, требует глубоких знаний и опыта
